# Engineering workflow

## Precedence

`AGENTS.md` and other repository instructions override this file.

## Planning

- Use a Conductor track for multi-step work.
- Obtain approval for a new track before implementation.
- Keep scope, non-goals, risks, and acceptance criteria explicit.

## Implementation

- Work one planned task at a time.
- Use existing repository patterns and required checks.
- Record actual validation commands and outcomes.
- Do not mark a task complete when validation is unknown or failing.

## Review and release

- Review completed work against its specification and plan.
- Record manual verification where it is required.
- Use normal Git and repository review procedures.
